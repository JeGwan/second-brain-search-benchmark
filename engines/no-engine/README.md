# no-engine — 데이터셋 자기검증 어댑터 (Dataset Self-Test)

`no-engine` 어댑터는 쿼리를 무시하고 `second_brain/` 의 **모든 문서를 그대로 반환**합니다.
전체 코퍼스가 검색 컨텍스트로 들어가므로, 핵심사실 커버리지가 **100% 여야 정상**입니다.

## ✅ 무엇을 검증하는가
이것은 검색 엔진이나 모델 능력을 재는 베이스라인이 **아닙니다.** 전체 문서를 줬는데도
어떤 `key_fact` 가 회수되지 않는다면, 그 사실은 검색 실패가 아니라 **질문지 정의가
코퍼스 텍스트와 어긋난 것**입니다. 즉 `no-engine` 은 `questions.json` 의
`key_facts`/알리아스/정규화 규칙이 실제 데이터셋과 정합하는지 확인하는 **셀프테스트**입니다.

## 실행
```bash
python3 evaluator.py run --engine no-engine
```
보고서의 **핵심사실 커버리지(헤드라인)** 가 **100%** 이면 질문지가 코퍼스와 정합합니다.

## 100% 미만이면 — 질문지 점검 신호
전체 문서를 반환했는데도 회수되지 않은 사실이 있다는 뜻이므로, 다음을 점검하세요.
1. 누락된 `key_fact` 의 알리아스가 코퍼스의 실제 표면형과 일치하는가
   (예: 계좌번호 `882-0192-332` ↔ `8820192332` 처럼 표기 차이는 알리아스로 흡수).
2. 정규화(NFKC·소문자·공백·천단위 콤마 제거) 후에도 매칭되지 않는 표기 차이가 있는가.
3. `reference_notes` 파일명이 실제 `second_brain/` 파일과 일치하는가(문서 재현율).

보고서의 "문항별 세부" 섹션이 어느 사실이 어떤 알리아스로 매칭됐는지/누락됐는지를
보여주므로, 누락 사실을 그대로 짚어 질문지를 수정하면 됩니다.
